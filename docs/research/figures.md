# Figure Design

## Overview

Figures are one of the most important tools for communicating research. A good figure helps readers quickly understand an idea, result, or pattern that might otherwise require substantial text to explain.

Some useful goals when designing figures:

* Readers should be able to understand the main message without relying heavily on the caption.
* A figure should communicate a clear idea, not simply display data.
* Main results figures should be able to stand on their own and tell an important part of the paper's story.
* Particularly successful introductory or contextual figures are often reused by others in presentations, lectures, policy briefs, and other communication materials.

There is rarely a single correct way to visualize information. The goal is not simply to display data, but to communicate clearly.

## Start with the Message

Before creating a figure, ask:

> What is the main thing I want the reader to learn?

Different figures serve different purposes, including:

* Summarizing data
* Presenting main results
* Exploring mechanisms or heterogeneity
* Providing context
* Building confidence in a result
* Explaining a concept or workflow

A figure should have a clear purpose and a clear takeaway.

## Prioritize Clarity

Readers should be able to understand the main point of a figure quickly.

Consider:

* Clear labels and titles
* Readable text
* Logical panel organization
* Direct annotations when helpful
* Consistent visual encodings

Whenever possible, avoid making readers search for important information.

## Keep Things Simple

Every visual element should have a purpose.

When reviewing a figure, ask:

> If I removed this, would the figure become harder to understand?

If not, consider removing it.

Simple figures are often more effective than complicated ones.

## Choose the Right Visualization

Different questions call for different plot types. Do not default to a particular chart simply because it is familiar.

When looking for ideas, it is often helpful to draw inspiration from:

* Scientific papers
* Data journalism
* Visualization galleries
* Previous work from the lab

Many effective figures are adaptations of ideas used elsewhere.

## Be Consistent

Within a project, related figures should feel related.

Consider using:

* Consistent color schemes
* Similar visual styles
* Common labeling conventions

Consistency makes projects easier to follow and reduces cognitive burden for readers.

## Be Honest

Figures should help readers understand the data, not exaggerate or obscure it.

Use scales, colors, labels, and visual encodings thoughtfully. Whenever practical, show the underlying data and provide enough context for readers to interpret results accurately.

## Iterate

Strong figures are rarely produced on the first attempt.

Experiment with different layouts, plot types, annotations, and color schemes. Small changes can substantially improve interpretability.

## Final Thought

A useful test is to imagine a future collaborator encountering the figure with little context.

Could they understand the main message without reading the caption?

For important figures, another useful question is:

> Would someone else want to borrow this figure for the introduction to a presentation on the topic?

The best figures make ideas easy to discover, understand, communicate, and reuse.
